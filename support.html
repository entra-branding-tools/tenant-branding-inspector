Sign-in Page Branding Checker

Support view

Some companies add their own styling to the Microsoft sign-in page. This tool checks that styling for tricks that could move or stack images on top of the login boxes — a technique attackers use to fool people.

Option A · Check a company by its ID

Use this if you only have the company's Tenant ID (a long code like 72f988bf-…) or its domain (like contoso.onmicrosoft.com). The tool tries to read the styling automatically. If your browser blocks that (which can happen), a short set of instructions will appear here to help you finish the check.

If the automatic check didn't work, do this:
  1. Click the sign-in link shown just above to open that company's sign-in page in a new tab.
  2. On that page, hold Ctrl and press U to show the page's underlying code (or right-click the page and choose View page source).
  3. Hold Ctrl and press F, type customcss, and press Enter to jump to it.
  4. Select and copy the full web address around it — it starts with https:// and contains the word customcss.
  5. Come back here, paste it into Option B, and click Check the styling.

Tip: if the search finds nothing, this company isn't using custom styling — there's nothing to check.

Option B · Check a styling link

Use this if you already have the styling web address (it contains the word customcss). This is the most reliable way to check.

This tool only reads and reports — it never changes anything. A finding means a setting could move or stack content on the sign-in page; it doesn't prove misuse. When in doubt, share the results with the security team.
